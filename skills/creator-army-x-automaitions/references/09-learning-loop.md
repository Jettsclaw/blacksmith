# The Learning Loop

This replaces traditional "kill/scale" testing. The old way fought Meta's
algorithm. This system works WITH it. Meta decides what works. You learn
from those decisions and feed smarter creative back in.

## The Core Principle

**Don't kill ads. Read them.**

If Meta doesn't spend on an ad, that IS the kill signal. You don't need to
do it manually. The ad sits there costing you nothing. If Meta does spend on
an ad, that's the signal to learn from.

Your job is not to manage the algorithm. Your job is to FEED it better
creative, informed by what it's already telling you.

## The Loop

```
LOAD → LISTEN → LEARN → CREATE → LOAD (repeat)
```

### 1. LOAD: Create and Load Diverse Creative

Produce 12+ genuinely diverse ads and load them into one ad set.
"Diverse" means each ad must differ on at least 3 of the 5 diversity
dimensions (see Creative Diversity section below).

Don't set expectations about which ads will win. Load them. Walk away.

### 2. LISTEN: Let Meta Decide (7-14 days)

Do NOT touch the ad set for at least 7 days. Let Meta's AI distribute
spend across the creatives. What happens:

- Some ads get significant spend → Meta found an audience
- Some ads get minimal spend → Meta tested and rejected
- Some ads get zero spend → Meta didn't even bother

**All three outcomes are data.** None of them require you to do anything
inside the ad set. No pausing. No budget adjustments. No manual intervention.

### 3. LEARN: Extract Insight Patterns

After 7-14 days, analyze what Meta chose to spend on. Ask these questions:

**Hook Pattern:**
- What hook TYPE did the winning ads use?
  (Question? Claim? Visual interrupt? Relatable scenario?)
- Was there a common emotion? (Fear? Curiosity? Humor?)
- Did sound-off or sound-on hooks perform differently?

**Body Pattern:**
- What body STRUCTURE did winners share?
  (Demo-led? Story-led? Comparison? Testimonial stack?)
- Which building blocks appeared in winners?
  (Problem → Solution? Failed Solution → Unique Mechanism?)
- What was the pacing? (Fast cuts? Single take? Split screen?)

**CTA Pattern:**
- What CTA style drove clicks?
  (Direct? Community-framed? Soft entry? Urgency?)
- Where in the video did conversions happen?
  (Early clickers vs late watchers?)

**Customer Type Pattern:**
- Which CUSTOMER TYPE converted?
  (Did "Olivia" ads outperform "Dani" ads?)
- Does this reveal which barrier/motivator is strongest?
- Should the next batch lean harder into the winning persona?

**Format Pattern:**
- Which CREATIVE TYPE won?
  (UGC? Founder story? Comparison? Tutorial?)
- Does the brand's audience prefer a specific format?
- Are there formats that consistently get zero spend?

### 4. CREATE: Build Next Batch From Insights

This is the critical step. You are NOT Frankensteining assets. You are
Frankensteining INSIGHTS.

**Wrong approach (old Frankenstein):**
Take hook from Ad A, body from Ad B, CTA from Ad C, splice together.
→ Meta sees this as a near-duplicate. Andromeda groups it with the originals.
→ No new distribution. Wasted effort.

**Right approach (insight Frankenstein):**
Extract WHY each element won, then create entirely new creative that carries
the same strategic DNA in a completely different execution.

**Example:**

```
LEARNING FROM BATCH 1:
- Winning hook type: Relatable question ("Does your boyfriend steal your X?")
- Winning body structure: Live demo with genuine reaction
- Winning CTA: Community-framed ("You guys asked for this")
- Winning customer type: "Over-Plucked Olivia"
- Winning format: UGC

BATCH 2 CREATIVE BRIEF:
→ New relatable question hook (different question, different scenario)
→ New live demo (different product angle, different person, different setting)
→ Community CTA (different community moment, different offer)
→ Still targeting Olivia's pain point (regrowth/restoration)
→ UGC format but DIFFERENT talent, different location, different visual style
→ = Genuinely new creative that Andromeda treats as distinct
→ = Carries the proven strategic DNA
```

### 5. LOAD AGAIN: Add to Existing Ad Set

Load the new batch ALONGSIDE the existing winners. Don't remove anything.

- Winning ads from Batch 1 continue running
- New ads from Batch 2 get tested against them
- Meta distributes spend across all of them
- The best creative survives regardless of which batch it came from
- The ad set gets stronger over time, not reset every week

## Creative Diversity: The 5 Dimensions

Meta's Andromeda system groups creatives that look too similar. When ads are
grouped, they share distribution instead of getting independent reach. This
kills the benefit of having multiple creatives.

To ensure each ad gets independent distribution, new creative must differ
from existing ads on **at least 3 of these 5 dimensions:**

### 1. TALENT
Different person on camera. Different face, voice, energy.
Even the same person in a completely different context can work.

### 2. SETTING
Different location or background. Kitchen vs outdoor vs car vs office.
A visual environment the algorithm hasn't seen for this brand.

### 3. FORMAT
Different creative type. If Batch 1 was heavy UGC, add comparison,
tutorial, founder story, or behind-the-scenes to Batch 2.
Reference `references/06-creative-types.md` for all 12 types.

### 4. VISUAL STYLE
Different colour grading, framing, pacing, editing rhythm.
Warm tones vs cool tones. Quick cuts vs single take. Close-up vs wide.
Even the same content shot differently reads as "new" to Andromeda.

### 5. HOOK MECHANISM
Different opening structure. Question hook vs statement vs visual
interrupt vs text-first vs ASMR vs controversy. The first 3 seconds
must be structurally distinct from other ads in the set.

### The 3-of-5 Rule

When generating new creative, tag each ad against these 5 dimensions and
confirm it differs from existing ads on at least 3. This is a hard requirement,
not a guideline.

| Dimension | Existing Winner | New Ad | Different? |
|-----------|----------------|--------|------------|
| Talent | Sarah, kitchen | Mark, car | ✅ |
| Setting | Kitchen, warm light | Car, natural light | ✅ |
| Format | UGC testimonial | Side-by-side comparison | ✅ |
| Visual style | Warm, slow pace | Cool, quick cuts | ✅ |
| Hook mechanism | Question hook | Bold claim hook | ✅ |
| **Score** | | | **5/5 ✅** |

## The Weekly Rhythm

**Don't touch the ad set during the week.** The rhythm is:

**Monday:** Review last 7 days. What did Meta spend on? Why?
Extract the insight pattern. Log it.

**Tuesday-Wednesday:** Create next batch using the insights.
Apply the 3-of-5 diversity check on every new ad.

**Thursday:** Load new creative into the ad set alongside existing.

**Friday-Sunday:** Hands off. Let Meta distribute.

**The following Monday:** Review again. The loop continues.

---

## What NOT to Do

- **Don't pause low-spending ads.** They cost nothing. They're already "killed."
- **Don't adjust budgets based on individual ads.** Budget is at the campaign level.
- **Don't splice assets from winners into Frankenstein edits.** Andromeda groups them.
- **Don't restart the ad set every week.** Continuity compounds learning.
- **Don't panic after 48 hours.** Meta needs 7+ days to properly allocate.
- **Don't remove winning ads to "make room."** There's no cap on ads per ad set.

## What TO Do

- **Read the spend data.** It's Meta telling you what works.
- **Extract the WHY.** Not which ad won — why it won.
- **Create NEW creative** carrying the winning strategic DNA.
- **Ensure genuine diversity** on 3+ of 5 dimensions per new ad.
- **Load alongside winners.** Don't reset. Compound.
- **Log the patterns.** Over time, you build a playbook specific to this brand.

## The Compounding Advantage

Week 1: 12 ads, no data. Pure excavation-driven creative.
Week 3: 18 ads (12 original + 6 new). Data shows question hooks win.
Week 5: 24 ads. Data shows UGC + demo bodies + community CTAs win for Olivia.
Week 8: 30+ ads. You know exactly which customer type converts, which hook
type stops the scroll, which body structure holds attention, and which CTA
drives action. Every new ad is precision-targeted at the intersection of
all four patterns.

Week 1 brief: "Here are 12 diverse angles to test."
Week 8 brief: "Here are 6 new ads that match your proven winning formula
in strategy but are completely fresh in execution. Based on your data,
these should outperform Batch 1 by 40%+."

That's the difference between a template and a system.

## The `/multiplier` skill as the production-side variation engine

The learning loop above operates at the **Meta-spend level** (what got spent
on, why, how to feed insights back into new creative). It's the strategic
layer.

The `/multiplier` skill in the AI Ad Lab suite operates at the **prompt-creation
level** — taking ONE winning NB2 prompt and generating 5-8 Andromeda-compliant
variations using the same strategic DNA but genuinely different visual scenes.

**The two work together:**

| Layer | Question it answers | Tool |
|---|---|---|
| Strategic (this file) | What pattern won and why? What angle should the next batch lean into? | CA reads spend data |
| Production (/multiplier) | Once we know the winning angle, how do we make 6 more ads that carry the same strategic DNA without near-duplicate penalty? | `/multiplier` runs the variation engine |

**Workflow integration:**
1. Run learning loop (this file) weekly — extract strategic insights
2. For the winning concepts identified, feed the winning NB2 prompt + Brand
   DNA + VOC into `/multiplier`
3. `/multiplier` produces 5-8 genuinely diverse variations
4. Add the variations to the existing ad set (per this file's "LOAD AGAIN"
   step — don't reset)
5. Meta distributes across all of them; the variations test against the
   original winners

The 5 diversity dimensions in this file (Talent / Setting / Format / Visual
Style / Hook Mechanism) are operationalised inside `/multiplier`'s variation
engine. Brief `/multiplier` once and it handles the diversity check
automatically per variation.

## Pre-spend QA — Attention AI batch selection (OPTIONAL, future expansion)

Before loading a new batch of creatives into the ad set, you CAN run them
through Attention AI for pre-flight winner selection. See `19-attention-ai.md`
Workflow D — the multi-ad comparison trick.

**Current scope:** OPTIONAL. Per `SKILL.md` Step 7.5, Attention AI is currently
mandatory for landing pages only while the operator learns the tool. Static-ad
batch validation is in the queue for when scope expands.

The trick (when you're ready to use it):
1. Combine 3-5 of your new creatives into ONE composite image
2. Run through Attention AI
3. See which one pulls the eye first
4. Re-order and re-run to confirm it's not positional bias
5. Launch the cross-reference-validated winners first; hold the others for
   later batches

This is **pre-flight winner selection without ad spend.** Pairs perfectly
with the LISTEN phase of this learning loop — Meta confirms the winner over
7-14 days; Attention AI predicts it in 5 minutes. When the two agree, you
have signal you can trust.
